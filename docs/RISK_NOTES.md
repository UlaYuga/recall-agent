# Constraints, риски и митигации

## Hard constraints (не обсуждается)

### Дедлайн
- **Submission:** 11 мая 2026, 16:00 МСК (9:00 AM ET)
- **Buffer перед сдачей:** последние 30 минут (15:30-16:00) - только заполнение submission формы и финальная проверка ссылок. Никакого кода в это окно.

### Команда и формат
- Соло-проект, без команды
- Все коммуникации через Discord хакатона
- Все секреты в `.env`, в репе только `.env.example` с заглушками

### API лимиты
- 50K credits Runway на хакатон (free tier для всех участников)
- Telegram Bot API: **50MB максимум** на файл через cloud bot
- Если файл > 50MB - локальный bot server (но для нашего 720p 30-45с не нужно)

### Visual constraints
- **Только motion graphics**, никакого realistic talking head
- **Никаких реальных лиц людей** (Runway moderation + репутационный риск iGaming)
- **Никаких логотипов** конкретных операторов или провайдеров (1xBet, Mostbet, Pragmatic Play и т.д.)
- **Никаких слотов/игр под брендом** - только generic abstract abstract slot reels, generic playing cards, generic dice
- Брендовая палитра - своя, нейтральная (deep purple/gold/magenta), не копирующая конкретного оператора

### Контентные правила сценария
- Никаких "гарантированных выигрышей"
- Никакого "не упусти шанс", "обязательно", "точно выиграешь"
- Срочность только один раз в скрипте (один deadline на offer)
- Тон теплый, как от знакомого менеджера
- Никаких упоминаний психологических triggers манипулятивного типа (FOMO давить нельзя)

## Риски и митигации

### Риск 1: Runway moderation флажит контент

**Вероятность:** средняя. iGaming сам по себе не запрещен, но если в визуалах будут конкретные слоты, реальные деньги, или скрипт упомянет gambling slang ("ставки", "крупный куш") - могут зафлажить.

**Митигация:**
- Все visual_brief на английском, generic abstract: "abstract slot reels, golden coins, cinematic motion"
- В скрипте используем нейтральные слова: "подарок", "бонус", "возвращайся", "у нас для тебя кое-что есть"
- В EN-версии лендинга для submission - вообще убираем gambling-контекст, позиционирование "Consumer Subscription Reactivation"
- Если флажит конкретный кадр - повторяем gen4_image с другим промптом, не тратя video credits
- **Бэкап:** EN-лендинг и EN-копия submission text никак не упоминают gambling, только generic AI agent for retention

### Риск 2: Telegram sendVideo фейлит

**Вероятность:** низкая для 720p 30-45с (~10-15MB), но возможна на сетях с прокси.

**Митигация:**
- Fallback: первый месседж - `sendPhoto(poster.jpg)` + inline кнопка "Посмотреть видео" ведет на лендинг
- Второй месседж - `sendVideo` (если не пройдет, остается работающая poster+ссылка)
- В demo video показываем оба варианта, преподносим как фичу, не как костыль

### Риск 3: ffmpeg склейка не работает / voiceover не синхронизирован

**Вероятность:** средняя. Это самый рисковый блок - сборка финального mp4.

**Митигация в порядке приоритета:**
1. Простой concat через demuxer на одинаковых codec (h264/aac), без re-encode
2. Если codec разный - re-encode через `-c:v libx264 -c:a aac`
3. Voiceover как отдельный audio track с `amix` фильтром поверх фоновой музыки (или вообще без фоновой музыки в первой версии)
4. Если синхронизация плывет - `atempo` filter для растягивания/сжатия аудио
5. **Крайний фолбек:** seedance2 (36 credits/sec) умеет длинное видео из reference image, без склейки. Дорого, но работает. Для 5-7 финальных видео хватит credits.
6. **Аварийный фолбек:** показываем 3-4 короткие сцены как отдельные мини-ролики на лендинге без аудио, voiceover отдельно как audio file под видео-плеером. Не идеально для submission, но рабочий demo.

### Риск 4: Runway rendering queue тормозит в выходные

**Вероятность:** высокая. Все хакатон-участники в одном пуле.

**Митигация:**
- Финальные видео для demo - **генерим в воскресенье вечером**, не в утро понедельника
- Параллелизм через `asyncio.gather` - не сериально, а одновременно 3-4 task
- В schedule секция 10 это уже учтено: 20:00-22:00 воскресенье - запуск финального батча
- Если очередь совсем встала - переключаемся на gen4_turbo (5 credits/sec), он быстрее и в очередь идет отдельно

### Риск 5: Не успеваем сделать все из чеклиста

**Вероятность:** высокая. 3 дня соло на полный pipeline - тонко.

**Митигация - cut list по приоритетам (что выкидываем первым):**
1. **Первое что режем:** RU-лендинг. Делаем после хакатона, на собеседования.
2. **Второе:** Email delivery (mailto stub оставляем, реальный SMTP не делаем)
3. **Третье:** Metrics dashboard - оставляем только big numbers + funnel, без ROI calculator interactive
4. **Четвертое:** Approval edit функционал - если не успеваем, оставляем только approve/reject, без inline edit script (для demo достаточно показать кнопки)
5. **Не режем:** end-to-end pipeline (trigger -> agent -> approval -> video -> telegram -> tracking), demo video, EN-лендинг

### Риск 6: Credits заканчиваются раньше срока

**Вероятность:** низкая при дисциплине, средняя если много экспериментов.

**Митигация:**
- В `backend/app/runway/client.py` встроить `track_usage()` который пишет в DB сколько потратили на каждую генерацию
- В CLI команда `make credits-check` - показывает остаток через `/v1/organization/usage`
- Hard rule: до субботы вечера используем только gen4_turbo для тестов (5 credits/sec), gen4.5 включаем только для финальных пробных видео в субботу 18:00+
- Если осталось < 8K credits в воскресенье вечером - не делаем 7 видео, делаем 5 ключевых

### Риск 7: Anthropic / OpenAI ключ не работает или rate limit

**Вероятность:** низкая.

**Митигация:**
- В `backend/app/agent/classifier.py` две функции: LLM call + rule-based fallback
- Rule-based fallback: простое decision tree на основе threshold по дням inactivity, total_deposits_amount, last_deposit_amount. Не идеально, но рабочий fallback.
- Скрипты для семи mock-игроков можно сгенерить заранее в субботу и закешировать в JSON. Тогда LLM в demo дергаем только для свежего player на live demo, остальные показываем pre-cached.

### Риск 8: Vercel/Railway деплой не поднимается в последний момент

**Вероятность:** средняя. Бывает что на пиковых нагрузках билд упадет.

**Митигация:**
- Деплоим **в воскресенье вечером**, не в понедельник утром
- В demo video основной фокус на работающем локальном демо со скринкастом, hosted версии - бонус
- Если хостинг лежит - в submission даем GitHub link + видео того, как все работает локально. Это не дисквалифицирует.

### Риск 9: Ноут падает / интернет валится в критический момент

**Вероятность:** низкая, но катастрофическая.

**Митигация:**
- Каждый этап - отдельный коммит в git, push сразу
- Demo video загружаем на YouTube unlisted **до** финального дня (хотя бы черновую версию в воскресенье вечером)
- Submission text лежит в Google Docs или iCloud Notes, не только локально
- Финальная папка `assets/demo_video/` тоже в git (или на отдельном Google Drive)

### Риск 10: Не получается зарегиться/попасть в Discord

**Вероятность:** низкая если регистрация заранее.

**Митигация:**
- Регистрируемся **сегодня-завтра**, не в день старта
- Email для регистрации использовать тот же что в Runway dev account
- Discord-инвайт прийдет на email, проверяем сразу

## Что делать если получишь блок аккаунта Runway

Сценарий: после submission или во время хакатона флажат проект из-за iGaming-контекста.

**Что у тебя остается независимо ни от чего:**
- GitHub репо со всем кодом (приватный твой, не отнять)
- Архитектура pipeline - твоя, не лицензированная Runway
- Mock event bus, agent prompts, approval dashboard, Telegram bot, лендинг, tracking - твой код
- Сгенеренные видео (mp4) - скачаны локально и в `assets/demo_video/`
- Demo video 2 мин на YouTube - твой канал

**Что нужно сделать чтобы перевыкатить:**
1. Перевыпустить ключи: `RUNWAYML_API_SECRET`, `TELEGRAM_BOT_TOKEN`, `ANTHROPIC_API_KEY`
2. Если Runway полностью забанил - swap на Veo / Kling / Luma / Pika через интерфейс `VideoProviderProtocol` в `backend/app/runway/client.py`. Замена ~1 час кода.
3. Лендинги перезаливать не нужно, бэкенд деплой - просто перезаливаем env vars

**Демо-кейс работает вечно** даже если Runway завтра вообще закроется - все mp4 уже сгенерены и лежат у тебя на руках.

## Финальная мантра дисциплины

- **Сон не режем.** Ночь субботы и воскресенья - спать. Соло-марафон без сна = код в мусорку и баги в demo.
- **Не делаем фичи которых не было в чеклисте.** Любая "а еще было бы круто..." откладывается на post-hackathon.
- **В понедельник до 13:00 - только сборка demo video.** Никакого нового кода в этот день. Если что-то не работает в 13:00 - оно не работает в submission, точка.
- **Каждый этап - коммит и push.** Локальный диск может умереть. Github не умрет.
